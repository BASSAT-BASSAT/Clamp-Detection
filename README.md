# Clamp Detection

YOLO-based object detection for clamp identification, trained on frames extracted from workshop video and annotated via Roboflow.

## Dataset

Training data (video, COCO export, YOLO dataset, and related files) is **not** stored in this repository. Download it from Google Drive:

**[Download dataset](https://drive.google.com/drive/folders/1-2gRkTrO4aAEHxRB9ziJwSFd5doToDPl?usp=sharing)**

After downloading, place the contents in the project root so the folder layout matches:

```
clamp detection/
├── Task.mp4                    # source video
├── frames/                     # extracted frames (optional if using COCO export)
├── Clamp Detection.coco/       # Roboflow COCO export
│   └── train/
│       ├── _annotations.coco.json
│       └── *.jpg
└── clamp_detection_yolo/       # optional — generated by train_yolo.py
    ├── data.yaml
    ├── images/
    └── labels/
```

## Setup

```bash
pip install ultralytics opencv-python torch
```

## Usage

### 1. Extract frames from video (optional)

If you only have `Task.mp4` and need raw frames:

```bash
python frame_extraction.py
```

Update `video_path` in `frame_extraction.py` if your video is not at the project root.

### 2. Train YOLO

Converts the COCO export to YOLO format and fine-tunes a model:

```bash
python train_yolo.py
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `--dataset-root` | `Clamp Detection.coco` | Path to Roboflow COCO export |
| `--output-root` | `clamp_detection_yolo` | Output YOLO dataset directory |
| `--model` | `yolo11m.pt` | Base YOLO weights |
| `--epochs` | `50` | Training epochs |
| `--imgsz` | `640` | Input image size |

Training outputs are saved under `runs/detect/`.

## Repository contents

| File | Description |
|------|-------------|
| `frame_extraction.py` | Extracts 30 evenly spaced frames from `Task.mp4` |
| `train_yolo.py` | COCO → YOLO conversion and training pipeline |
| `train_yolo.ipynb` | Notebook version of the training workflow |
| `clamp_detection_yolo/data.yaml` | YOLO dataset configuration |
